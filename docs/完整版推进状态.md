### 本机 Linux 签字阻塞（2026-06-13 二次实测）

| 检查项 | 结果 |
|--------|------|
| `docker --version` | **未安装**（`CommandNotFoundException`） |
| `wsl --status` / `wsl -l -v` | **未安装**（提示 `wsl --install`） |
| `bash deploy/scripts/run-production-signoff.sh small` | **未执行** — 无 bash + Docker 全栈环境 |

**解除阻塞**：在 Linux 测试机或 Gitee `docker-e2e` Runner 上执行签字；本机可跑 `production-acceptance.ps1 -SkipWhiteBox` 作预检。

### CI 加固（2026-06-13）

- `docker-e2e` 在 small 栈签字前执行 `validate-large-tier-compose.sh`（compose 干跑）。
- `docker-e2e` 增加 `only: tags`，发布打 tag 后可手动触发留档。
- 归档路径：`ci-docker-e2e.log`、`validate-large-tier-compose.log`、`production-*.txt`、`signoff-*.txt`、`mvn-test-latest.log`。

### prepare 热路径（2026-06-13）

- `CachedPolicy` 缓存 `taskId`/`taskCode`，策略缓存命中时跳过 `llm_ai_task_def` 查询（生产 P95 阈值不变）。

## 验收链对照

| 环境 | 脚本 | 阶段 |
|------|------|------|
| Windows 本地 | `production-acceptance.ps1` | 白盒 → 黑盒 → **SSO** → 链路 → 压测 |
| Linux Docker | `production-acceptance.sh` | 白盒 → *IT → e2e → **SSO** → journeys → 压测 |
| 一键签字 | `run-production-signoff.sh` | compose up + wait + `production-acceptance.sh` |

门禁摘要：`GATE-WB` · `GATE-BB` · **`GATE-SSO`** · `GATE-CH` · `GATE-ST`

## Linux 测试机下一步

1. **克隆并 small 栈签字**
   ```bash
   git clone https://gitee.com/zestcc/zest-llm.git && cd zest-llm
   bash deploy/scripts/run-production-signoff.sh small
   ```
2. **确认 GATE-SSO**：`sso.enabled=false` 时 SSO 阶段 SKIP；启用 ZestSSO 后重跑 `bash deploy/scripts/sso-smoke.sh`，浏览器步骤见 `deploy/scripts/sso-browser-checklist.md`。
3. **手动触发 Gitee `docker-e2e`**，归档 `deploy/test-reports/signoff-*.txt`。
4. **（可选）medium / large**：Langfuse profile → `run-production-signoff.sh medium`；Large Tier 单独排期。
5. **Helm 试部署**：`deploy/helm/zest-llm/README.md`，核对 Ingress 与 Secret。
