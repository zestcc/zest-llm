# 智能体探测与运维手册

> 适用版本 1.0.0+ · 对应 Admin 运维中心与智能体配置页

## 1. 智能体可用性探测

### 1.1 探测项

| 检查项 | 说明 | 关键 |
|--------|------|------|
| prompt-published | 已发布 Prompt | 是 |
| model-route | 有效模型路由 | 是 |
| profile-json | Profile JSON 合法 | 是 |
| policy-resolve | 策略可完整解析 | 是 |
| gateway-health | Provider `/health/liveliness` | 是 |
| provider-preset / outbound-secret | 预设与 SecretRef | 否 |
| mcp:* | MCP Server 工具列表 | 否 |
| smoke-invoke | 网关最小 chat（可选） | 否 |

### 1.2 整体状态

- **READY**：所有关键项通过  
- **DEGRADED**：关键项通过，存在非关键失败  
- **UNAVAILABLE**：任一关键项失败  

### 1.3 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/admin/agent-profiles/{taskCode}/probe` | 检测已发布 Profile |
| POST | `/api/admin/agent-profiles/{taskCode}/versions/{version}/probe` | 检测指定版本 |
| GET | `.../probe/latest` | 最近一次探测 |
| GET | `.../probe/history?page=&size=&profileVersion=` | 历史分页（可按版本筛选） |

请求体（可选）：

```json
{ "appKey": "order-service", "smokeTest": false }
```

### 1.4 配置

```yaml
zest-llm:
  admin:
    agent-probe:
      schedule-enabled: true          # 定时巡检
      cron: "0 */30 * * * ?"          # 默认每 30 分钟
      smoke-test-on-schedule: false   # 定时任务不做冒烟
      alert-webhook-url: http://alert-mock:8080/webhook
      alert-on-degraded: false        # 仅 UNAVAILABLE 告警（默认）
      alert-cooldown-minutes: 60      # 同状态冷却时间
```

## 2. Webhook 告警

探测落库后，若状态为 **UNAVAILABLE**（或开启 `alert-on-degraded` 时的 DEGRADED），且配置了 `alert-webhook-url`，会 POST JSON：

```json
{
  "type": "AGENT_PROBE",
  "taskCode": "aiChat",
  "profileVersion": "v1",
  "overallStatus": "UNAVAILABLE",
  "probeId": 123,
  "message": "..."
}
```

告警记录查询：`GET /api/admin/agent-probe-alerts?page=&size=&taskCode=`

## 3. 运维中心

Admin UI → **运维中心**（`/ops`）

| Tab | 能力 |
|-----|------|
| 成本告警 | FinOps 日成本超阈值 Webhook 记录 |
| Execution 归档 | 热表/冷表统计、立即归档、上次归档结果 |
| 智能体告警 | 探测 Webhook 发送记录 |

Dashboard 智能体健康：`GET /api/admin/dashboard/agent-health`

## 4. FinOps 成本告警

配额表 `llm_app_quota` 字段：

- `daily_cost_limit`：日成本上限  
- `alert_threshold_pct`：触发阈值（默认 80%）  
- `alert_webhook_url`：Webhook 地址  

每次 Runtime invoke 后若当日成本 ≥ 限额 × 阈值，发送 Webhook 并写入 `llm_cost_alert`（同一 App 每日仅一条）。

查询：`GET /api/admin/cost-alerts?page=&size=&appKey=`

## 5. Execution 归档

```yaml
zest-llm:
  admin:
    execution-archive:
      enabled: true
      retention-days: 90
```

- 统计：`GET /api/admin/executions/archive/stats`  
- 手动归档：`POST /api/admin/executions/archive/run`  

## 6. Langfuse 可观测

### 6.1 启用

```bash
docker compose -f deploy/docker-compose.yml \
  -f deploy/docker-compose.observability.yml \
  --profile observability up -d
```

Admin 环境变量：`SPRING_PROFILES_ACTIVE=docker,observability`

### 6.2 Execution 详情 Trace 链接

配置 UI 基址（可与 ingest API 不同）：

```yaml
zest-llm:
  admin:
    observability:
      langfuse-ui-base-url: http://localhost:3000
      trace-path-template: /trace/{traceId}
```

Execution 详情 API 返回 `observabilityTraceUrl`，Admin UI 可一键跳转 Langfuse。

配置查询：`GET /api/admin/config/observability`
