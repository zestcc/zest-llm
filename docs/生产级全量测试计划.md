# ZestLLM 生产级全量测试计划

> 版本：**2.0.0** · 对应 AC1–AC53 · 四阶段验收：**白盒 / 黑盒 / 链路 / 压测**

## 1. 测试目标

在接近生产的 Docker / 本地环境中，对 Control Plane、Runtime API、Admin UI、ZestFlow、Demo 进行**端到端**验证，确保功能、安全、性能、可靠性满足**生产上线**标准。

## 2. 四阶段验收矩阵

| 阶段 | 类型 | 覆盖 | 自动化入口 | 环境 |
|------|------|------|------------|------|
| **1 白盒** | 单元 + 可选 IT | SPI/Infra/Admin 逻辑、Controller 契约 | `mvn test` · `*IT`（Testcontainers） | 任意 |
| **2 黑盒** | API / 安全 / 功能 | SEC/API/RT/DEMO/PROBE/AC39–53、SPA | `full-acceptance.ps1` | Admin :8088 |
| **3 链路** | 业务旅程 | Admin 向导 → prepare → Demo methodA/flowChat | `demo-walkthrough.ps1` · `run-journeys.sh` | Admin + Demo + LiteLLM |
| **4 压测** | 性能门禁 | prepare 并发 P50/P95、成功率 | `stress-test-prepare.ps1` · `loadtest-cp-prepare.sh` | Admin :8088 |

**一键编排**：

```powershell
# Windows（无 Docker，Tier=local 阈值）
powershell -File deploy/scripts/production-acceptance.ps1

# Linux / Docker 全栈（Tier=production，P95≤500ms）
bash deploy/scripts/production-acceptance.sh
```

报告目录：`deploy/test-reports/production-*.txt` · `report-*.txt` · `mvn-test-latest.log`

## 3. 黑盒用例分级（full-acceptance）

### P0 — 阻断发布

| ID | 内容 |
|----|------|
| SEC-01~02 | 错误 Token、未登录 Admin 拒绝 |
| RT-01~03 | prepare / invoke / execution 审计 |
| CHAIN-01~02 | prepare→invoke→execution 全链路 |
| DEMO-01~02 | methodA / flowChat |
| REGISTRY-01~02 | ZestFlow Hub 注册与 peers |
| PROBE-01 | 探测执行 |
| PERF-01 | prepare P95（local≤800ms，生产≤500ms） |
| AC43 | Eval 发布门禁 |

### P1 — 核心治理

- API-01~25 Admin 模块可读
- AC29–38 探测 / Dashboard / Ops
- AC45–53 能力栈 / 模板 / Wizard / Learning

### P2 — 扩展

- AC16–25 Flow / Eval / 缓存（Docker e2e）
- `run-journeys.sh` 场景旅程

## 4. 环境要求

### 4.1 Windows 本地（无 Docker）

| 组件 | 端口 | 必需 |
|------|------|------|
| MySQL 8 | 3306 | 是 |
| Admin + CP | 8088 | 是 |
| LiteLLM mock | 4000 | invoke/Demo |
| Demo | 8081 | DEMO / CHAIN |
| MCP mock | 9090 | 可选 Tool Loop |

```powershell
powershell -File deploy/scripts/start-local-full.ps1 -EmbedUi -WithLiteLLM -WithDemo
powershell -File deploy/scripts/production-acceptance.ps1
```

### 4.2 Linux Docker 生产级

| 组件 | 端口 | 必需 |
|------|------|------|
| compose 全栈 | 见 deploy/docker-compose.yml | 是 |
| ZestFlow | 20551/20552 | AC16/17 |
| Valkey / Langfuse | 6379/3000 | medium+ |

```bash
cd deploy && docker compose up -d --build
bash deploy/scripts/production-acceptance.sh
```

## 5. 生产质量门禁（签字表）

| 门禁 ID | 指标 | local | production |
|---------|------|-------|------------|
| GATE-WB | `mvn test` | 0 FAIL | 0 FAIL |
| GATE-WB-IT | `*IT` Testcontainers | SKIP | 0 FAIL |
| GATE-BB | `full-acceptance` / `e2e-acceptance` | 0 FAIL | AC1–38 全 PASS |
| GATE-CH | demo-walkthrough / journeys | PASS | PASS |
| GATE-ST | prepare P95 | ≤800ms | ≤500ms |
| GATE-ST-R | 成功率 | ≥95% | ≥99% |
| GATE-SEC | P0 安全用例 | 100% | 100% |
| GATE-DB | Flyway | ≥ V22 | ≥ V22 |
| GATE-FEAT | `agentProbeApi=true` | 是 | 是 |

**签字**：执行人 · 日期 · 环境 · MasterReport 路径 · 遗留风险（S1/S2 须为 0）

## 6. 执行流程

```powershell
# Windows 完整四阶段
powershell -File deploy/scripts/production-acceptance.ps1

# 仅黑盒+压测（已跑过 mvn test）
powershell -File deploy/scripts/production-acceptance.ps1 -SkipWhiteBox

# 生产阈值（本地仍无 Docker e2e）
powershell -File deploy/scripts/production-acceptance.ps1 -Tier production
```

```bash
# Linux Docker 完整四阶段
TIER=production bash deploy/scripts/production-acceptance.sh

# 跳过 Testcontainers
SKIP_IT=1 bash deploy/scripts/production-acceptance.sh
```

## 7. 缺陷管理

| 严重级别 | 定义 | 处理 |
|----------|------|------|
| S1 | 数据丢失、认证绕过、生产不可用 | 阻断发布，24h 内修复 |
| S2 | 核心 AC/P0 失败 | 修复 + 全量回归 |
| S3 | 非关键 UI、文档偏差 | 排期修复 |

## 8. 回归策略

- **每次发布**：GATE-WB + GATE-BB（P0 全绿）
- **探测/运维变更**：+ PROBE + AC29–38
- **大版本 / 生产签字**：`production-acceptance.sh` 四阶段全绿 + 人工签字表
