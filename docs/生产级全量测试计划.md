# ZestLLM 生产级全量测试计划

> 版本：1.0.0 · 对应 AC1–AC38 与 `deploy/scripts/full-acceptance.ps1`

## 1. 测试目标

在接近生产的 Docker / 本地环境中，对 Control Plane、Runtime API、Admin UI、ZestFlow、Demo 进行**端到端**验证，确保功能、安全、性能、可靠性满足上线标准。

## 2. 测试范围矩阵

| 维度 | 覆盖内容 | 自动化入口 |
|------|----------|------------|
| 功能 | AC1–38、Admin 全模块 API、Runtime invoke/prepare/stream | `e2e-acceptance.sh` + `full-acceptance.ps1` |
| 安全 | 错误 Token、未登录 401/403、OPERATOR 只读 | `full-acceptance.ps1` SEC-* |
| 性能 | prepare P50/P95（默认 30 样本，P95≤800ms 本地） | `loadtest-cp-prepare.sh` / PERF-* |
| 可靠性 | 探测落库、归档运行日志、Webhook 重发 | API-09~16、PROBE-* |
| 兼容性 | SPA 深链、8088 直出 UI | COMPAT-* |
| 单元 | SPI/Infra/Admin 服务逻辑 | `mvn test` |

## 3. 环境要求（生产级 E2E）

```bash
cd deploy && docker compose up -d   # mysql valkey litellm admin demo
# 或 Windows: .\deploy\scripts\local-dev.ps1 + IDEA local profile
```

| 组件 | 端口 | 必需 |

## 4. 测试用例分级

### P0 — 阻断发布

- AC6 错误 Token → AUTH_FAILED
- AC2 Admin 登录 + Execution 追溯
- AC29–31 探测执行与历史
- AC38 meta/features + 新 probe API
- SEC-02 未授权 Admin 拒绝
- PERF-01 prepare P95 在阈值内（生产建议 ≤500ms）

### P1 — 核心治理

- AC3 Prompt 热更新、AC7–10 Profile/Provider/OIDC
- AC11–15 MCP / SSE
- AC32–37 运维中心、批量巡检
- API-01~25 Admin 模块可读

### P2 — 扩展能力

- AC16–25 Flow / Eval / 缓存
- AC33–36 观测与 FinOps
- Journeys（`run-journeys.sh`）

## 5. 执行流程

```powershell
# 1. 单元测试
mvn clean install
mvn test -pl zest-llm-admin -Dtest='*Test'

# 2. Windows 全量 Admin 验收
powershell -ExecutionPolicy Bypass -File deploy/scripts/full-acceptance.ps1

# 3. Linux/Docker 完整 AC（需 compose 全栈）
bash deploy/scripts/e2e-acceptance.sh
bash deploy/scripts/run-journeys.sh
bash deploy/scripts/loadtest-cp-prepare.sh
```

## 6. 缺陷管理

| 严重级别 | 定义 | 处理 |
|----------|------|------|
| S1 | 数据丢失、认证绕过、生产不可用 | 阻断发布，24h 内修复 |
| S2 | 核心 AC 失败、探测/归档不可用 | 修复 + 回归 |
| S3 | 非关键 UI、文档偏差 | 排期修复 |

缺陷记录模板：`ID | 模块 | 复现步骤 | 期望 | 实际 | 级别 | 状态`

## 7. 回归策略

- 每次发布前：P0 全绿 + `mvn test` 全绿

- 大版本：Docker 全栈 + `e2e-acceptance.sh` + `run-journeys.sh`
