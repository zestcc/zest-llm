# ZestLLM 生产级验收标准

> 版本 1.0.0 · 不以分期里程碑划分，全部条目为发布门禁。

## 1. 功能验收（AC1–AC6）

| ID | 场景 | 步骤 | 通过标准 |
|----|------|------|----------|
| AC1 | Demo 同步 AI | Compose 启动后调用 Demo | 返回 `answer` + `traceId` |
| AC2 | Execution 查询 | Admin UI / API 按 traceId 查 | 可见 input/output/token |
| AC3 | Prompt 热更新 | Admin 发布新版本，不重启 Demo | 下次调用使用新版本 Prompt |
| AC4 | SPI Noop | `observability: noop` | invoke 仍成功 |
| AC5 | 自动化旅程 | `deploy/scripts/e2e-acceptance.sh` + CI | 全部 PASS |
| AC6 | 鉴权失败 | 错误 appToken 调用 Runtime | `errorCode=AUTH_FAILED` |
| AC7 | Profile 下发 | prepare 返回 `profileVersion` + `gatewayBaseUrl` | 字段非空 |
| AC8 | Provider 切换 | Admin activate-provider | 下次 prepare 使用新 Provider |
| AC9 | Profile 导入 | Admin import JSON + publish | 版本可见且 PUBLISHED |
| AC10 | OIDC 配置 | App Auth 绑定 OIDC_JWT JSON | Admin API 可读写 |
| AC11–AC36 | MCP / Stream / ZestFlow / Playground / Eval / FinOps / Agent Probe / Ops | `deploy/scripts/e2e-acceptance.sh` | 全部 PASS |

执行：

```bash
cd deploy && docker compose up -d --build
bash deploy/scripts/e2e-acceptance.sh
mvn -pl zest-llm-admin -am test
```

## 2. 架构验收

| 项 | 标准 |
|----|------|
| 三段式 Runtime | `/v1/llm/prepare` · `/invoke` · `/report` 可用 |
| 治理不分流 | 推理流量不经 CP 转发 token |
| SPI 可替换 | 改 `zest.llm.adapters.*` 配置，零业务代码变更 |
| 策略缓存 | Prompt/Route 变更后立即 `invalidate` |
| 数据库 | 仅 MySQL 8 |

## 3. SPI 生产实现清单

| SPI | 生产实现 | 配置键 |
|-----|----------|--------|
| ModelGateway | LiteLLMGatewayAdapter | `model-gateway: litellm` |
| Observability | LangfuseObservabilityAdapter / Noop | `observability: langfuse\|noop` |
| PolicyCache | ValkeyPolicyCacheAdapter / Caffeine | `policy-cache: valkey\|caffeine` |
| ResponseCache | ValkeyResponseCacheAdapter / Noop | `response-cache: valkey\|noop` |
| AlertWebhook | HttpAlertWebhookAdapter / Noop | `alert-webhook: http\|noop` |
| Quota | RedisTokenBucketQuotaAdapter / Db | `quota: redis-token-bucket\|db` |
| Audit | JdbcAuditAdapter / 内置 | `audit: jdbc` |
| SecretResolver | Env + Vault + Composite | `env:` / `vault:path#key` |
| McpToolAdapter | HttpMcpToolAdapter (JSON-RPC) | Profile tools type=mcp |
| ContentModeration | KeywordBlocklistModerationAdapter | `content-moderation: keyword-blocklist` |
| PromptRenderer | HandlebarsPromptRenderer | `prompt-renderer: handlebars` |
| OutputSchema | JsonOutputSchemaValidator | `output-schema-validator: json` |

## 4. Admin 治理面

| 模块 | 要求 |
|------|------|
| 租户 | 列表 + 创建 |
| 应用 / 作业 / Prompt / 路由 | CRUD + 发布 |
| Execution | 分页 + traceId 详情 |
| 配额 | App 级日 Token / QPS |
| 审计 | 操作留痕 + 查询页 |
| 方法注册 | Starter 上报元数据 |
| 智能体配置 | Profile 版本 + Provider 预设 + Auth + import/export + **可用性探测/历史** |
| 运维中心 | FinOps 成本告警 + Execution 归档 + 智能体 Webhook 告警 |
| 适配器健康 | 全 SPI 探测 |
| Dashboard | 统计 + 成本图表 + **智能体健康/告警** |
| Playground | Prompt 预览 + Admin 内 Invoke |
| Eval | 数据集批量 invoke + 通过率报表 + CRUD |
| FinOps | 日成本阈值 Webhook 告警 |
| Execution 归档 | 热表保留 + 冷归档表 + Admin 统计 |

## 5. 质量门禁

| 项 | 标准 |
|----|------|
| 单元测试 | 鉴权、缓存失效、Runtime 错误契约 |
| 集成测试 | Testcontainers MySQL + Mock Gateway |
| CI | `mvn test` 必须通过 |
| 合规 | LICENSE + NOTICE + THIRD_PARTY_LICENSES |

## 6. KPI

| 指标 | 目标 |

| traceId 覆盖 | 100% |
| CP P95（不含推理） | < 50ms |
| fallback 成功率 | > 95% |
